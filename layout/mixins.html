# mixins的混合方法和混合数据

全局的方法和全局的数据是必不可少的。这样可以大量增强复用性,让代码量可以很大程度上减少。

在global文夹中定义的mixins需要手动引入到index.js中。然后index.js会自动向外暴露。所有的方法将会在初始化的时候混合到全局。这些通用的形式希望不需要每次都手动引入。而其它针对于个别项目的mixins的选项希望进行手动引入。因为这样可以让项目的维护者可以更好的理解方法和数据的来源。

注意

所有的混合数据和事件名都在前面加上_代表是私有的，这样大大避免在合并策略的同名的时候被覆盖。

# 项目全局的混合方法

弹窗页面的混合方法是项目的必用的一些方法。无论是打开弹窗页面还是关闭弹窗页面,我们不需要引入任何东西。只需要直接调用混入的全局方法就可以。全局方法暂时暴露出两个方法

# _openDialog

this._openDialog({
  name: '指定的文件路径'
})

# _closeDialog

this._closeDialog()

在弹窗页面中如果执行完某个方法之后。自动关闭弹窗页面的时候，直接可以手动调用this._closeDialog()方法。手动关闭当前对应的弹窗

# 基于element封装的混合方法

element ui给我们提供了很多方法。但是对于方法如果每次都去重复写一次。就会显的非常沉余,通过封装之后在mixins进行混入,不但不需要进行引入。同样我们简化代码量。进行操作。同样所有对element ui有特别想法的封装方法统一可以写在此文件中。

# _submitForm

当form表单提交的时候，需要对rules规则进行校验的完毕之后再写入回调方法。通过promise封底装。可以很好的用async await进行融合的简化操作

methods: {
  async hander () {
    await this_submitForm()
    console.log('表单规则检测通过')
  }
}

# _confirmDelete

当对某些行为需要再次确认的时候,可以调用_confirmDelete方法。封装的原形是根据this.$confirm进行通过promise二次封装。可以很好的用async await进行融合的简化操作

methods: {
  async hander () {
    await this_confirmDelete()
    console.log('确认完毕,可以执行操作')
  }
}

# _confirmMessage

当某些行为完成之后，要提示对应的消息和状态，可以调用_confirmMessage方法，原形是通过this.$message进行封装

methods: {
  async hander () {
    await this_confirmDelete()
    this._confirmMessage('你已经确认了')
  }
}
